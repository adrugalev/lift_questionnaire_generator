# Генератор опросных листов EPSS

Streamlit-приложение заполняет Excel-опросный лист по группам лифтов и сохраняет исходную русско-китайскую структуру шаблона. Пользователь может ввести данные вручную или загрузить ТЗ в PDF/DOCX, проверить найденные значения и скачать готовый XLSX.

## Быстрый запуск

```powershell
cd C:\Users\Drugalev\Documents\Codex\2026-07-07\files-mentioned-by-the-user-xlsx\outputs\lift_questionnaire_generator
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
streamlit run app.py
```

Excel-шаблон вшит в приложение: используется `templates/questionnaire_template.xlsx`, созданный из приложенного опросного листа.

## Как работать

1. Откройте приложение Streamlit.
2. Если есть ТЗ заказчика, загрузите PDF/DOCX в левой панели.
3. Приложение само выберет режим: при загруженном ТЗ заполнит найденные поля, без ТЗ будет работать как ручная форма.
4. Проверьте и дополните данные проекта: заказчик, название объекта, адрес, дата заполнения.
5. Добавьте нужное количество групп лифтов и заполните параметры.
6. Нажмите `Сгенерировать опросник` и скачайте Excel-файл.

## Excel mapping

Маппинг строк вынесен в `data/excel_mapping.json`. В текущем шаблоне найден лист `Новосергиевка`, диапазон строк 1-57, русские подписи в колонке A, китайские подписи в колонке B, данные групп лифтов начиная с колонки C.

Ключевые строки:

- `project_name` -> row 1
- `section` -> row 2
- `lift_name` -> row 3
- `quantity` -> row 4
- `capacity_kg` -> row 5
- `speed_ms` -> row 6
- `stops` -> row 8
- `doors_count` -> row 9
- `cabin_type` -> row 14
- `door_opening_type` -> row 27
- `machine_room` -> row 42
- `mgn_accessibility` -> row 51

Колонки A/B не перезаписываются. Название проекта записывается в строку 1 каждой колонки группы, начиная с C.
Строки 52 `Стоимость` и 53 `Количество контейнеров` приложение не заполняет: эти значения остаются пустыми для расчета заводом.

## Справочники

Предзаполненные значения selectbox находятся в `data/options.json`. Если в интерфейсе выбрать `Другое...` и ввести новое значение, оно будет добавлено в соответствующий справочник.

## Версия

Под заголовком приложения выводится подпись вида `Версия N от DD.MM.YYYY`. Логика находится в `src/version.py`.

## Парсинг ТЗ и PDF

MVP поддерживает:

- DOCX через `python-docx`
- PDF через `pypdf`
- табличные PDF через `pdfplumber`
- встроенный OCR через `rapidocr-onnxruntime` и `pymupdf`, если PDF является сканом/чертежом
- дополнительный OCR через `pytesseract`, если установлен системный Tesseract
- OpenAI vision fallback через `OPENAI_API_KEY`, если локальный OCR недоступен или не справился
- название объекта и адрес из штампа чертежного PDF, если они доступны OCR-слою
- извлечение строк вида `параметр -> значения по группам лифтов`
- базовое извлечение регулярными выражениями: проект, грузоподъемность, скорость, остановки, двери, размеры кабины, двери, приямок, верхний этаж

DOC-файлы нужно предварительно конвертировать в DOCX или PDF.

Для сканированных PDF есть несколько вариантов распознавания.

Вариант 1: встроенный локальный OCR. Он работает через `rapidocr-onnxruntime` и не требует отдельной установки Tesseract.

Вариант 2: системный Tesseract OCR. Одного Python-пакета `pytesseract` недостаточно: на компьютере должен быть установлен Tesseract OCR и доступен в `PATH`.

Windows:

1. Установите Tesseract OCR.
2. Добавьте папку с `tesseract.exe` в `PATH`.
3. Проверьте в терминале:

```powershell
tesseract --version
```

Если OCR недоступен, приложение покажет предупреждение и не будет придумывать отсутствующие значения.

Вариант 3: OpenAI vision. Укажите ключ API в переменной окружения:

```powershell
$env:OPENAI_API_KEY="..."
```

После этого при PDF без нормального текстового слоя приложение попробует распознать страницы как изображения и вернуть строгий JSON по модели `Questionnaire`.

## Ограничения MVP

- Распознавание ТЗ не является полным инженерным NLP-парсером.
- DOC без конвертации не читается.
- OCR для сканов требует установленный Tesseract OCR.
- LLM/vision-извлечение вынесено в `src/llm_extractor.py` и включается только при наличии `OPENAI_API_KEY`.
- Приложение не переводит русские значения на китайский, а сохраняет китайские подписи шаблона.
